Cal Fire tracker

Background

The Chronicle has a California Fire Tracker, one of the most viewed pages ever on the site. The information is manually retrieved and inputted from the Cal Fire website. Before my internship, Cal Fire changed its website and removed the RSS feed that the Chronicle team had used to send automated Slack alerts when updated. Instead, fire updates were now added to a CSV file on the site.

What I did

I wrote a Python script that connected to a Slack webhook my supervisor set up to check the CSV url every 10 minutes and notify the team if anything changed. This was quite a challenge for me, as I had not used much Python before. My first version partly worked, then my supervisor suggested a different way which I implemented and it turned out to be much more reliable.

What I learned

This relatively small script was very frustrating because I couldn’t always immediately tell why certain things didn’t work, but ultimately rewarding because it fixed a workflow issue for the digital team.
